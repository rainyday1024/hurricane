## 핵심 원칙

1. **읽기 순서**: `CLAUDE.md` → `AGENTS.md`(이 문서) → 관련 `docs/`.
2. **추측 금지**: 모호하면 질문. 확실하지 않은 채로 코드 작성하지 않는다.
3. **유사 기능 우선 검색**: 새로 만들기 전 기존 구현 확인. 같은 패턴/스타일로 작성.
4. **공통화 제안**: 중복 패턴 발견 시 작업 전에 사용자에게 제안.
5. **검증 통과**: 작업 중 빠른 검증 반복 확인, **완료 시 완료 검증 통과 필수**. 실패하면 통과할 때까지 수정(실패 상태로 종료 금지).
6. **PR/Push/삭제는 명시 지시 후에만**: 자동으로 하지 않는다.
7. **민감정보 금지**: 비밀번호/키/토큰을 코드/커밋에 절대 포함하지 않는다.
