# FEEDBACK_LOOPS.md — 피드백 루프 / 검증 자동화

> 에이전트(및 개발자)가 실수를 **즉시** 깨닫게 하는 자동 피드백 체계의 현황.
> 결정이 바뀌면 이 문서를 갱신한다.

## 0. 왜 필요한가

`CLAUDE.md` / `AGENTS.md` 의 규칙은 **읽고 해석해야 적용**된다 — 어겨도 자동으로 막히지 않는다.
린터·타입 체커·테스트·CI 같은 피드백 루프는 규칙을 위반하는 **순간 빨간불**을 띄워,
"글로 쓴 규칙"을 "강제되는 규칙"으로 바꾼다.
피드백 루프가 없으면 "규칙은 많은데 채점자가 없는" 상태가 된다.

## 1. 현재 상태

> <!-- 자기 프로젝트의 실제 상태로 채운다. "있는 척" 금지 — 없으면 없다고 적는다. -->

| 피드백 루프 | 상태 | 비고 |
|-------------|------|------|
| 컴파일 / 타입 검사 | <✅ / ❌> | <어느 명령에 포함되는지> |
| 린터 / 정적분석 | <✅ / ❌> | <도구명 또는 미적용 사유> |
| 테스트 스위트 | <N개 / 0개> | <위치·프레임워크> |
| pre-commit hook | <✅ / ❌> | <설치 여부> |
| CI/CD · PR 검증 | <✅ / ❌> | <파이프라인 정의 위치 또는 없음> |

> <!-- 테스트가 0개·정적분석이 없다면, 검증 명령은 사실상 "컴파일(+패키징) 되는가" 수준임을 명시. -->

## 2. 결정 사항

- **검증 명령**: 빠른 검증(작업 중 반복) = `<...>`, 완료 검증(완료 기준) = `<...>`.
- <정적분석 도구 도입/미도입 결정과 사유>
- <pre-commit hook 사용/미사용 결정 — 다른 문서가 훅 존재를 전제하지 않도록 여기서 못박는다>

## 3. 지금 에이전트가 지켜야 할 것

- 작업 중에는 빠른 검증으로 반복 확인하고, **완료 시 반드시 완료 검증 통과**를 확인한다.
  실패하면 통과할 때까지 수정한다(`AGENTS.md` 종료 체크리스트).
- **한계 인지**: <예: 테스트가 없어 런타임 오류·회귀는 자동으로 잡히지 않는다 → UI/동작 변경은
  `UI_VALIDATION.md` 수동 절차를 반드시 병행>.
