## 스펙 문서 작성 양식

```markdown
# <모듈명> (<코드>)

## 개요
<무엇을 하는 모듈인가>

## 주요 화면 / URL
| URL | 설명 | 컨트롤러/핸들러 |
|-----|------|-----------------|

## 데이터 모델
| 테이블/엔티티 | 설명 |
|---------------|------|

## 비즈니스 규칙
- ...

## 의존
- 다른 모듈/외부 시스템 연동
```

## 빈 항목 채우기 가이드

- 코드를 처음 만지는 모듈은 **수정 전에** 이 디렉터리에 스펙 파일을 먼저 만들고
  알게 된 사실을 기록하면서 작업한다.
- 기존 코드와 다른 동작이 발견되면 스펙 문서를 갱신한다.
