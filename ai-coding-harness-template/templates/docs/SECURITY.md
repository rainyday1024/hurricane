# SECURITY.md

> 보안 규칙. OWASP Top 10 기준으로 작성.

## 절대 금지

- 비밀번호/API 키/토큰 평문 커밋
- `.env`, 설정 파일(`application-*.yml` 등)의 시크릿 평문 커밋
- 사용자 입력 그대로 SQL/HTML/Shell 에 삽입

## 인젝션 (SQL / Command)

- DB 파라미터는 **반드시 바인딩(prepared statement)** 사용. 문자열 연결로 쿼리를 만들지 않는다.

       동적 컬럼명 등 불가피한 경우만 화이트리스트 검증 후 허용. -->
- 셸/외부 프로세스 호출 시 사용자 입력을 인자로 그대로 넘기지 않는다.

## XSS

- 템플릿 출력은 기본 escape 사용. raw/unescaped 출력은 입력 신뢰원을 명시하고 코드 리뷰 표시.
- 클라이언트에서 `innerHTML` 등에 사용자 입력을 넣을 때 반드시 escape.

## 인증 / 인가

- **주 메커니즘**: <예: 세션 기반 / 토큰(JWT) — 설정 위치 명시>.
- **메서드/엔드포인트 단 권한 체크**: <프레임워크의 권한 애노테이션/미들웨어>. 새 엔드포인트에서 누락 금지.
- **서버 검증 필수**: 클라이언트 검증만으로 끝내지 않는다 (필수값·형식·상태·권한을 서버에서도 검증).
- 새 인증 방식(JWT 도입 등) 추가 전 사용자 확인 필수.

## 비밀 관리

- 시크릿은 환경 변수 또는 외부 비밀 저장소. 저장 시 암호화.
- 운영 비밀을 리포지토리에 두지 않는다.

## 파일 업로드

- 확장자/크기/MIME 검증. 업로드 경로는 웹 루트 외부, 정적 서빙은 권한 통과 핸들러 경유.

## API 문서 노출

- API 문서(Swagger 등) 자동 생성 도구의 **운영 환경 노출 여부**를 사용자에게 확인.
- 외부 노출 API 는 인증/CSRF 정책을 명시적으로 정의.

## 실시간 통신 (WebSocket 등)

- 메시지 핸들러에 권한 체크 누락 금지.

## 의존성

- 업그레이드 시 사용자 승인 필수. CVE 발견 시 즉시 보고.

## 로그

- 비밀번호/토큰/개인식별정보(주민번호 등)는 **절대 로그에 남기지 않는다**. 사용자 식별자는 마스킹.
- SQL/요청 로깅 도구가 바인딩 값을 노출하면 운영 환경에서 레벨 조정.
