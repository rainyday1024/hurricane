## 권장 MCP 목록 (예시 — 프로젝트에 맞게 취사선택)

### 1. 이슈 트래커 (읽기 전용 권장)

- **용도**: 이슈 조회, 본문/상태/담당자/코멘트 확인. 브랜치명·plan 작성 시 컨텍스트 확보.
- **권장 흐름**: 사용자가 이슈 번호를 언급하면 AI 가 먼저 조회하고 본문을 요약해 인식 일치 확인 후 작업.
- **절대 금지 (읽기 전용 정책 시)**: 상태 변경·진척률 수정·코멘트/첨부·새 이슈 생성 등 **모든 쓰기성 호출**.
  사용자가 "이슈에 X 해줘" 라고 지시해도 정책에 따라 거절하고, 사용자가 직접 하도록 안내.

### 2. 개발 DB (읽기 위주)

- **용도**: 개발 DB 스키마/데이터 직접 조회. 매퍼 작성 전 컬럼명·타입 확인, 쿼리 결과 확인.
- **⚠️ 주의**
  - **개발 DB 임. 운영 DB 가 아님.** 다른 개발자도 사용하므로 변경 영향 고려.
  - 읽기 쿼리(`SELECT`)는 자유롭게.
  - **쓰기 쿼리(`INSERT`/`UPDATE`/`DELETE`)는 사용자 명시 지시 후에만.**
  - DDL(`CREATE`/`DROP`/`ALTER`)은 절대 금지. 스키마 변경은 사용자가 직접.

### 3. 브라우저 자동화 (예: Playwright)

- **용도**: UI 동작 검증 (`UI_VALIDATION.md` 절차). 페이지 진입 → 시나리오 → 스크린샷.
- **금지**
  - 사용자 자격증명(아이디/비밀번호) 임의 입력 금지. 로그인 필요 시 사용자에게 요청.
  - 운영/스테이징 URL 자동 접속 금지. localhost 또는 사용자가 명시한 URL 만.

## 추가 참고

- 세션 시작 시 어떤 MCP 가 로드됐는지 모르겠으면 도구 목록으로 확인.
- MCP 가 로드되지 않았으면 사용자에게 알리고, 해당 단계는 "수동 확인 필요" 로 처리.
