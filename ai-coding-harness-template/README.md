# AI 코딩 하네스 스타터킷 (AI Coding Harness Starter Kit)

> Claude Code 등 **AI 코딩 에이전트가 팀의 규칙·구조·컨벤션대로** 일하게 만드는
> 문서 기반 "하네스(harness)" 템플릿. 어떤 프로젝트에도 복사해서 채워 쓸 수 있습니다.

이 저장소는 실제 운영 중인 대규모 엔터프라이즈 프로젝트에서 다듬은 하네스 체계를
**프로젝트 특화 정보를 제거하고 재사용 가능한 템플릿**으로 정리한 것입니다.

---

## 왜 필요한가

AI 에이전트에게 "코드를 짜라"고만 하면, 프로젝트의 규칙·구조·맥락을 몰라 제멋대로 작성합니다.
아키텍처를 어기고, 중복 코드를 만들고, 위험한 패턴을 쓰고, 매번 다른 스타일로 짭니다.

**하네스 = 에이전트를 우리 방식대로 일하게 하는 가이드레일의 묶음**입니다.
사람 신입에게 온보딩 문서·코드 리뷰·CI를 주듯, 에이전트에게도 똑같이 줍니다.

| 기둥 | 의미 | 이 템플릿의 파일 |
|------|------|------------------|
| ① 문서화 (컨텍스트) | 프로젝트 규칙·구조·도메인 지식 | `CLAUDE.md`, `AGENTS.md`, `docs/*` |
| ② 아키텍처 제약 | 레이어링·모듈 경계·금지 패턴 | `docs/ARCHITECTURE.md` |
| ③ 피드백 루프 | 실수를 즉시 잡는 자동 검증 (린터·테스트·훅) | `docs/FEEDBACK_LOOPS.md` |
| ④ 생명주기 관리 | 빌드·배포·PR 워크플로 | `docs/GIT_WORKFLOW.md` |

> 핵심 통찰: 문서(①②)는 "읽고 해석해야" 적용되지만, 피드백 루프(③)는 어기는 **순간 빨간불**이 켜집니다.
> 둘 다 필요합니다.

## 저장소 구조

```
ai-coding-harness-template/
├── README.md          # ← 이 문서 (소개 + 빠른 시작)
├── GUIDE.md           # 하네스를 처음부터 구축·이식하는 방법론 (내러티브)
├── LICENSE            # MIT (저작권자 이름만 채워 쓰세요)
├── templates/         # 그대로 복사해 채워 쓰는 템플릿
│   ├── CLAUDE.md          # 진입점 + 운영 규칙 (안정 문서, 앵커)
│   ├── AGENTS.md          # 60줄 이하 업무 지침 (안정 문서, 앵커)
│   └── docs/              # 카테고리별 상세 가이드
│       ├── ARCHITECTURE.md    # 레이어·모듈 경계·금지 패턴
│       ├── PLAN_SYSTEM.md     # 비단순 작업의 계획(plan) 절차
│       ├── FEEDBACK_LOOPS.md  # 검증 자동화 현황·결정
│       ├── GIT_WORKFLOW.md    # 브랜치·커밋·배포
│       ├── UI_VALIDATION.md   # UI 변경 검증 절차
│       ├── SECURITY.md        # 보안 규칙 (OWASP 기준)
│       ├── TOOLS.md           # MCP / 외부 도구 사용 규칙
│       ├── DOC_GARDENING.md   # 문서 유지보수 거버넌스 (메타)
│       └── product-specs/
│           └── index.md       # 모듈/도메인 스펙 카탈로그
```

`templates/` 안의 문서에는 `<이렇게>` 각괄호로 표시된 **채워야 할 자리**와
`<!-- 예: ... -->` 형태의 **작성 예시 주석**이 들어 있습니다. 자리를 채우고 예시 주석은 지우세요.

## 빠른 시작 (도입 5단계)

1. **`templates/` 를 대상 프로젝트 루트로 복사** — `CLAUDE.md`, `AGENTS.md` 는 루트에,
   `docs/` 는 원하는 이름(예: `claude-docs/`)으로.
2. **핵심 3~4개부터 작게 시작** — `CLAUDE.md` · `AGENTS.md` · `ARCHITECTURE.md` 의
   `<플레이스홀더>`를 먼저 채웁니다. 한 번에 전부 채우려 하지 마세요(→ [GUIDE.md](GUIDE.md) §2).
3. **코드와 대조** — AI 로 초안을 만들면 추측이 섞입니다. 실제 코드·설정을 열어 사실을 교정하세요
   (빌드 명령, 프레임워크 버전, 훅 존재 여부 등).
4. **거버넌스 합의** — `docs/DOC_GARDENING.md` 의 규칙(SSOT·안정문서·분량·편집 전 점검)을 팀이 먼저 합의합니다.
5. **점진 확장** — 실제 작업하며 필요한 문서를 늘리고, 반복되는 실수는 해당 문서에 규칙으로 추가합니다.

전체 방법론·타임라인·이식 가이드는 **[GUIDE.md](GUIDE.md)** 를 참고하세요.

## 이 하네스가 겨냥하는 도구

문서 기반이라 특정 도구에 묶이지 않지만, 다음 파일명은 널리 쓰이는 관례를 따릅니다.

- **`CLAUDE.md`** — [Claude Code](https://claude.com/claude-code) 가 세션 시작 시 자동으로 읽는 프로젝트 지침 파일.
- **`AGENTS.md`** — 여러 AI 코딩 도구가 공통으로 인식하는 에이전트 지침 파일 관례.

다른 에이전트를 쓴다면 해당 도구가 읽는 진입 파일명에 맞춰 조정하세요. 나머지 `docs/` 는 그대로 재사용됩니다.
