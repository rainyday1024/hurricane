# 건설공사 원가계산 시스템 — 작업 내역

> 기술 스택: Spring Boot 2.7 · Java 17 · MyBatis · Oracle · Pebble · AUIGrid/SlickGrid

건설공사 단가자료(노무·자재·기계경비·일위대가 등)를 등록·관리하고 원가계산 결과를 산출하는 시스템의 백엔드·프런트엔드 기능을 담당했습니다. 주요 기능은 **Oracle 기반 본 시스템과 PostgreSQL 이식 버전 양쪽**에 구현했으며(이중 DB 대응), 이는 Oracle PL/SQL과 PostgreSQL SQL을 각각 작성·검증하는 작업을 포함합니다. 아래는 담당·개발한 모듈과 주요 기능을 정리한 것입니다.

## 2. 주요 개발·개선 내역 (2026)

### 원가계산 핵심 로직
- **원가계산 제비율** 규모구간 정비 — 일반관리비·이윤·고용보험·간접노무비·기타경비의 공사규모별 요율 체계 추가·변경
- **일위대가 손료 계산/재계산** — 비목별 분배 로직, 소수점 처리(0.1원 미만 버림 후 합산), C3R 파일 업로드 시 손료 자동 재계산 및 합계 갱신
- **전체계산 단가 오류 수정** — 표준시장단가 0원 산출 버그 등

### 원가계산 사용성 개선
- 자료 **찾아가기** 팝업 및 Context Menu(찾아가기·새로고침) 기능
- **표시순서 드래그** 방식 개편 + Context Menu 표시순서 변경
- 산정근거 **소계·총계** 버튼, 경비항목/일위대가 하위항목 **미리보기**
- 변경내역서 작성완료 처리 및 표시순서 정책 정비

### 물가지·자재 자동화
- **물가지-자재 자동매핑** — 매핑 스코어 정규화, 자동매칭 중단 기능, 저장 방식 이관(stateless)
- **물가지 정보 자동적용** — 인도조건 세그먼트 파생, 저장 키 정비, 기준월(물가적용년월) 기준 일원화
- **물가지 정보 검색** — 그룹핑 해제 조회, 원본가 표시, 유사도 기반 필터링, 소수 단가(유류 등) 표시, 검토 필요 안내

## 3. 이중 DB (Oracle / PostgreSQL) 구현

원가계산·물가지·자재·크롤링 등 핵심 기능은 **Oracle 본 시스템**과 **PostgreSQL 이식 버전** 두 저장소에 각각 반영했습니다. PostgreSQL 쪽은 단순 복제가 아니라 Oracle 전용 문법(PL/SQL, DECLARE 블록, `DECODE`/`NVL`/`CONNECT BY` 등)을 PostgreSQL SQL로 재작성·검증하는 작업이 포함됩니다.

- PostgreSQL 버전에서도 `@author 김성욱` 표기 자바 파일 **28개**, 고유 작업 커밋 **약 48건** 보유
- 대표 항목: 물가지 자동적용/정보검색, 원가계산 기능계산, 웹수집단가·API 연계이력 관리, 일위대가 손료 재계산, 표준WBS 적용기준관리, 자재단가 조회 성능개선

## 4. 활동 규모 (참고)

커밋 수는 브랜치 흐름(feature → develop → release)에서 체리픽·머지로 중복 계상되기 쉬워, **머지 커밋과 브랜치 간 중복을 제거한 "고유 작업 커밋"** 기준으로 집계했습니다.

| 저장소 | 고유 작업 커밋 |
|--------|:---:|
| Oracle 본 시스템 (eurecax2) | 약 110건 |
| PostgreSQL 이식 (eurecaxpg) | 약 48건 |

*집계 범위는 현재 git 이력(2026-02~)에 한정됩니다. 파일 헤더상 실제 담당은 2023년부터이며(표준시장단가·사용자관리 등), 2023~2025년 작업은 저장소 재초기화로 현재 git 이력에 포함되지 않습니다. 따라서 위 수치는 실제 기여도를 과소평가합니다.*
